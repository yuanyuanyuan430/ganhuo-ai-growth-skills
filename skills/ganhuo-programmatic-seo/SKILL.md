# 干活 程序化 SEO

## Overview

Use this skill to make programmatic SEO boring, bounded, and testable before scaling.

## Workflow

1. Validate that the long-tail pattern has repeatable search demand and distinct user value.
2. Define the entity set, fields, data sources, and page-level uniqueness requirements.
3. Create 10 sample pages first; review whether they answer materially different questions.
4. Specify template modules: unique intro, data table, comparison, local/category context, FAQ, internal links, and update timestamp.
5. Plan crawl/index controls: sitemap, canonical, noindex rules, pagination, duplicate handling, and GSC monitoring.
6. Only scale after samples show indexation, impressions, and no obvious quality pattern failure.

## Output

Return pSEO feasibility verdict, field schema, sample-page checklist, scale gates, and monitoring plan.

## References

Read `references/source-map.md` when you need source provenance, material-specific guardrails, or the original file list behind this skill.

## Boundaries

- Do not produce thousands of thin pages before validating samples.
- Do not hide duplicate pages with technical tricks while keeping no user value.
