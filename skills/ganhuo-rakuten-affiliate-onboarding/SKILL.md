# 干活 乐天联盟注册

## Overview

Use this skill to prepare a truthful Rakuten Publisher application and avoid mixing demo values with real account data.

## Workflow

1. Confirm the applicant's country, entity/person type, website URL, niche, traffic sources, and contact information.
2. Check website readiness: live URL, original content, policy pages, contact path, affiliate disclosure, no placeholder sections.
3. Prepare profile inputs from real data only: audience, channel, promotion methods, and traffic range.
4. Flag tax/payment fields that require user/legal/accounting confirmation.
5. For advertiser applications, match niche, geography, content format, traffic source, and channel restrictions.
6. Write short advertiser application notes that explain the site, audience, and promotion plan without exaggeration.

## Output

Return an onboarding checklist, missing inputs, advertiser-fit notes, and application wording draft.

## References

Read `references/source-map.md` when you need source provenance, material-specific guardrails, or the original file list behind this skill.

## Boundaries

- Do not invent identity, traffic, tax, or payment information.
- Do not treat tutorial screenshots as values to copy into a real application.
